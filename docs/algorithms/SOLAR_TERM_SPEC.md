# Solar Term & Pillar Calculation Spec (Draft)

## 목적
- 절기(24절기) 기준 월주 계산의 경계 조건을 명확히 정의
- 양력 입력에서 연/월/일/시주 산출을 위한 입력/출력 요구사항을 정리

## 입력 요구사항
- 양력 생년월일 (YYYY-MM-DD)
- 출생시간 선택 (HH:mm)
- 타임존 (기본 Asia/Seoul)

## 절기 기반 월주 규칙(개요)
- 월주는 **절기 기준**으로 구분
- 경계: 특정 절기의 정확한 시각 이전/이후로 월주가 달라짐
- 예: 입춘 이전 → 전년도 절기 체계, 입춘 이후 → 새 연도 기준

## 계산 단계(개요)
1. 입력을 타임존 기준으로 표준화
2. 해당 날짜/시간에 대응하는 **절기 경계** 계산
3. 절기 경계를 기준으로 월주 산정
4. 천간/지지 60갑자 순환으로 연/월/일/시주 산출

## 경계 조건 체크리스트
- 절기 경계 시각과 동일한 시각 입력 시 처리 규칙 명확화
- 출생시간 미상 시 월주/시주 처리 규칙
- 윤년, 윤달(음력 변환이 필요 시) 처리 여부 명확화

## 경계 시각 처리 규칙(결정)
- 절기 경계 시각과 **동일한 시각** 입력은 **새 절기 적용**으로 처리
- 절기 경계 **이전**은 이전 절기 기준 월주 유지
- 절기 경계 **이후**는 다음 절기 기준 월주 적용

## 입춘 기준 연주 변경 규칙(결정)
- 연주는 **입춘 시각을 기준**으로 변경
- 입춘 이전 출생: 전년도 연주 적용
- 입춘 이후(동시각 포함) 출생: 해당 연도 연주 적용

## 예시 케이스

### 예시 1: 입춘 직전
- 입력: 2024-02-04 07:59 Asia/Seoul
- 규칙 적용: 입춘 이전 → 전년도 연주 적용

### 예시 2: 입춘 동시각
- 입력: 2024-02-04 08:00 Asia/Seoul
- 규칙 적용: 입춘 동시각 → 해당 연도 연주 적용

### 예시 3: 절기 경계 동시각
- 입력: 절기 경계 시각과 동일
- 규칙 적용: 새 절기 기준 월주 적용

### 예시 4: 하지 직전/직후
- 입력: 2010-06-21 23:59 Asia/Seoul (하지 직전)
- 규칙 적용: 하지 이전 → 이전 절기 기준 월주

### 예시 5: 동지 직전/직후
- 입력: 2015-12-22 00:30 Asia/Seoul (동지 직후)
- 규칙 적용: 동지 이후 → 다음 절기 기준 월주

### 예시 6: 월주 변경 케이스(절기 경계 전후)
- 입력: 절기 경계 전 1분
- 규칙 적용: 이전 절기 기준 월주 유지

### 예시 7: 월주 변경 케이스(절기 경계 동시각)
- 입력: 절기 경계 시각 정확히 일치
- 규칙 적용: 새 절기 기준 월주 적용

## 검증 전략
- KASI 등 공공 절기 데이터와 비교
- 공개 절기표와 결과 대조
- 골든 픽스처로 일관성 검증
- 입춘과 월주 변경 절기는 경계 전 1분, 경계 동시각, 경계 후 1분을 테스트한다.
- 현재 내장 절기표는 `fixture-limited` 상태로 관리하며, KASI 원자료 재검증 완료 전에는 넓은 공개 베타 범위나 넓은 날짜 지원의 근거로 사용하지 않는다.

## 산출물
- `calculatePillars(input)`에서 절기 계산 모듈 분리
- 경계 시각 데이터는 캐시 가능

## Phase 2A 구현 범위
- 현재 계산 코어는 `Asia/Seoul`만 지원한다.
- 절기 경계 데이터는 fixture와 경계 테스트에 필요한 연도부터 내장한다.
- 입춘 동시각은 새 연주/월주 적용으로 처리한다.
- 출생시간이 없고 입력일이 입춘 또는 월주 변경 절기 경계일이면, 정오 등 임의 시각으로 추정하지 않고 시간 입력이 필요하다는 에러를 낸다.
- 월주는 12개 월 경계 절기(입춘, 경칩, 청명, 입하, 망종, 소서, 입추, 백로, 한로, 입동, 대설, 소한)를 기준으로 한다.
- 23:00 이후 자시를 다음 날짜 일주로 넘기는 옵션은 아직 구현하지 않는다.
- 지원 범위 밖 날짜는 잘못된 값을 조용히 반환하지 않고 에러를 낸다.

## Phase 2D 경계 테스트 보강
- 2024-02-04 입춘 경계는 17:26, 17:27, 17:28을 테스트한다.
- 2024-03-05 경칩 월 경계는 KASI 기준 11:22, 11:23, 11:24를 테스트한다.
- 경계 동시각과 경계 이후 1분은 새 절기 기준으로 처리한다.
- 경계 이전 1분은 이전 절기 기준을 유지한다.
- 현재 MVP는 23:00 이후 자시를 다음 civil date의 일주로 넘기지 않는다. 예: 2010-06-21 23:00은 2010-06-21의 일주를 유지하고 시지만 자시로 계산한다.
- 23:00 이후 일주 rollover를 지원하려면 별도 제품 결정, 사용자 안내, fixture 재검증, 호환성 테스트가 필요하다.

## Phase 2E fixture matrix 확장
- 2024년 월주 변경 절기 중 청명, 입하, 망종, 소서, 입추, 백로, 한로, 입동, 대설의 경계 전 1분, 경계 동시각, 경계 후 1분을 테스트한다.
- 2024년 경계일에 출생시간이 없으면 월주를 정오 등으로 추정하지 않고 `birthTime is required` 에러를 유지한다.
- 2024-12-07 대설 이후의 월 경계 상한 검사를 위해 2025-01-05 소한과 2025-02-03 입춘 경계만 내장한다.
- 이 단계는 넓은 연도 범위의 절기 데이터 ingestion이 아니라, 검증 가능한 2024년 fixture matrix를 추가하는 제한 범위다.
- 공개 베타 전에는 내장 절입 시각을 KASI 원자료와 다시 교차 검증한다.
- Phase 4S는 `docs/SOLAR_TERM_SOURCE_AUDIT_2026-05-25.md`에 내장 경계값, 현재 출처 상태, KASI 재검증 체크리스트를 기록한다. 이 문서는 재검증 준비 자료이며, 재검증 완료 증거가 아니다.
- Phase 4T는 `docs/KASI_SOLAR_TERM_REVALIDATION_2026-05-25.md`에 KASI 2024/2025 역서 대조 증거를 기록했다. Phase 4U는 1분 차이 행을 KASI 기준으로 맞췄으며, 이 검증 범위는 내장 2024 matrix와 2025 상한 경계에 한정된다.
