# Refund And Support Draft

Status: Draft for paid-launch readiness. This is not a finalized legal policy.

This draft defines the minimum refund and support boundaries needed before Saju Lab opens checkout for a one-time detailed report.

## Support Scope

Support should cover:
- payment failure
- duplicate charge
- paid report download failure
- corrupted or unreadable PDF-ready export
- missing required notices in the paid export
- user questions about data handling or report scope

Support should not promise:
- a changed interpretation because the user dislikes the result.
- professional investment, medical, legal, or psychological advice.
- deterministic prediction accuracy.
- account recovery before accounts exist.

## Refund Draft

Refunds should be considered when:
- checkout succeeds but the paid report cannot be generated.
- the user is charged more than once for the same report.
- the paid export cannot be downloaded due to a product issue.
- required disclaimers or scope notices are missing from the paid output.

Refunds may be declined when:
- the report was successfully generated and downloaded.
- the request is based only on disagreement with interpretation tone or content.
- the user asks for professional advice outside the product scope.

## Response Expectations

Before checkout opens, define:
- support contact address or form.
- expected response window.
- what information users should include.
- how order identifiers are handled.
- how support messages are retained or deleted.

Placeholder support path:
- `support@example.com` must be replaced before live checkout.

## Failed Download Handling

Before live checkout:
- the product should explain how users can retry a download.
- support should be able to identify a paid order without requiring the user to send birth data in plain text.
- support should redact birth data, calculated pillars, report text, and PDF-ready HTML if the user voluntarily sends them.
- if a regenerated report is needed, the data-retention decision must state whether regeneration is local, server-side, or hybrid.
